4. Naming Decision Guide

When naming an element, follow this decision flow:

// Step 1: What file is this in?
// ‚Üí view/sale/return_request.tpl
// ‚Üí prefix: sale-return_request

// Step 2: What IS this element?
// ‚Üí It's a button that saves the form
// ‚Üí element: save-btn

// Step 3: Does it belong to a logical group?
// ‚Üí The "process" section has: bar, step, icon, label, line
// ‚Üí All prefixed with "process-"

// Step 4: Does it have states?
// ‚Üí step can be: done, active, upcoming
// ‚Üí modifier: --done, --active

// Result:
.sale-return_request__process-step--done
